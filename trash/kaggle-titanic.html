解决matplotlib中文乱码问题

参考资料：彻底解决matplotlib中文乱码问题

  1. 查找matplotlib配置文件目录 
     import matplotlib
 print  matplotlib.matplotlib_fname()
    假设输出得到：/usr/local/lib/python2.7/dist-packages/matplotlib/mpl-data/matplotlibrc
  2. 从windows的c:\Windows\Fonts目录下将SimHei（即黑体）的字体文件拷贝到linux的/usr/local/lib/python2.7/dist-packages/matplotlib/mpl-data/fonts/ttf目录下
  3. 修改配置文件/usr/local/lib/python2.7/dist-packages/matplotlib/mpl-data/matplotlibrc
    找到font.family和font.sans-serif，取消注释，并为font.sans-serif字段添加SimHei
     font.family    : sans-serif        
 font.sans-serif : SimHei, Bitstream Vera Sans, Lucida Grande, Verdana, Geneva, Lucid, Arial, Helvetica, Avant Garde, sans-serif
    找到axes.unicode_minus字段，取消注释并将其值改为False（解决“-”显示方块的问题）
  4. 清空matplotlib缓冲目录~/.cache/matplotlib

数据集初探


根据官网对数据的描述：Titanic-Data，各字段含义如下：

字段 描述 备注
Survived 存活与否的标志 0 = 未存活，1 = 存活
Pclass 乘客等级 分为1、2、3三等（1等最高级）
Sex 性别 male = 男，female = 女
Age 年龄
Sibsp 兄弟sibling、配偶spouse的数量
Parch 父母parent、孩子children的数量
Ticket 船票编号
Fare 票价
Cabin 船舱号
Embarked 登船的港口 C = Cherbourg，Q = Queenstown，S = Southampton

PassengerId只是一个序号，没有什么意义；
Survived、Pclass、Age、Sibsp、Parch、Fare都是数值型的数据；
Name和Ticket、Cabin都是一些文本数据；
Sex、Embarked是离散型的数据，但并非数值，可以在处理的时候离散为0、1等数值

总体信息


数据集共有891组数据，并且具有12个属性
数据集中Embarked仅有两个数据缺失，Age也有一部分缺失，而Cabin数据缺失比较严重

基本特征描述

  • 存活率为38.4%；
  • 年龄范围从0到80，平均年龄约29.7；
  • 超过一半的人没有兄弟、配偶，超过75%的人只带了一个兄弟、配偶，但最多的居然有8个兄弟、配偶；
  • 超过75%的人没有带父母、孩子，但最多的也带了有6个父母和孩子；
  • 票价则是0-512不等，75%的人票价都低于31，然而最高票价居然高达512.3，土豪任性……

属性分布

Jupyter - 属性分布


  • 获救人数比未获救人数的一半再多一点
  • 等级分布上，三等的人数远远高于一等和二等，而一等和二等的人数相近
  • 性别分布上，男性要远远多余女性
  • 年龄主要集中在20-40区间
  • 绝大多数人没有带兄弟、配偶或是父母、孩子，也就是说孑然一身，不过带一个兄弟或配偶、带一个父母或孩子的人也不少
  • 票价绝大多数都低于100，但最高的几个票价高的厉害
  • 绝大多数人都是登船港口都是S的

获救情况在各属性上的分布

Jupyter - 获救情况在各属性上的分布
再来看看各个属性分布跟获救与否的关系


乘客等级跟获救与否有明显的关系，可以看到一等的获救比例超过了一半，二等的获救比例也接近一半，而三等的获救比例甚至不到四分之一


乘客性别也跟获救与否有明显的关系，女性的获救比例超过了四分之三，lady-first


年龄跟获救与否似乎也有一定的联系；
从年龄上看，0-8岁的孩子获救比例很高，应该是“女人、小孩先走”；
其次是16-40岁之间的青壮年，应该是体能比较好的一批人吧；


兄弟、配偶的数量上看，如果有一两个兄弟、配偶的人获救率似乎要高一些；
可能是相互帮助、相互照应吧，但是人数如果太多了也反而降低了存活率；


情况与兄弟、配偶数量类似；


票价高于30的乘客存活率非常高！
看来票价高的乘客所活动的区域的应急设施可能比较齐全，或者登上救生艇的时候优先选择了出价高的乘客；


C港口登船的乘客似乎存活率稍微高一些

数据处理

jupyter - 数据处理

  • 将非数值的二分类特征数值化
    • 性别Sex分为male(0), female(1)
    • 舱位Cabin分为无数据(0), 有数据(1)
  • 将多分类的特征划分为多个二分类特征
    比如将 乘客等级(1,2,3) 划分为 是否是等级1(0,1) ，是否是等级2(0,1) ， 是否是等级3(0,1) 三个特征
    登船港口Embarked也可以如此操作；
    另外，性别Sex、舱位Cabin也可以直接这么划分，这样一来数据缺失的特征也会被考虑进去
  • 数据缺失处理
    • 如果缺失的数据不多，可以考虑直接抛弃缺失数据
    • 离散值可以使用上一点的划分方式，或者直接把“缺失”作为一个新类别
    • 连续值
      • 补特殊值，如0
      • 补平均值
      • 补拟合值
      • 把连续值按一定步长分段离散化，然后按缺失离散值处理
  • 连续值的归一化处理

改进1

jupyter - 改进1

措施

  • Sex、Embarked改用dummy的方式
    不再简单地以0，1，2去区分，同时保留数据缺失的特征
  • 对Age采用分段处理，不用拟合，每10岁一档，保留数据缺失的特征
    以下是各年龄段的分布情况，总体数量上还是比较平衡的

效果

没有很明显的效果，正确率没有明显的变化——

但由于不采用随机森林算法对年龄进行拟合，学习速度提高了不少

各属性权重

我们来看看各属性的权重，可能可以看出一些端倪

  • SibSp和Parch属性明显处理的不好
    如前所述，不携带兄弟配偶、父母子女的存活率不及携带一两个的，但是携带太多反而成为累赘
    所以可以进一步地把SibSp和Parch分为0、1或2、大于2三组比较合理
  • Fare系数为正，说明有钱人存活率高呀
  • Cabin数据缺失较多，而且都是一些不是很能理解的字符串，难以处理
    但可以看到Cabin有数据比没数据的存活率明显更高
  • 从Embarked上看
    C、Q港口似乎对存活率的影响不大
    S港口却明显降低存活率
  • 从Pclass上看
    一等的存活率明显较高，三等的存活率明显较低，二等似乎没啥影响
  • 从Sex上看
    female的系数为正，而且很高；male的系数则为负；lady first的绅士风度很明显
  • 从Age上看
    年龄缺失好像没什么影响
    0-10岁的孩子存活率明显较高
    10-30岁的乘客存活率适中
    30-40岁的乘客存活率反而高一些emmm可能是体力还不错，而且有一定生活阅历——临危不惧吧？
    40岁以上就不行了，尤其是50岁以上的乘客，年龄的负面影响挺大的
